# Contributing

Contributions are welcome. Please open an issue before submitting a pull request so the change can be discussed first. Keep pull requests focused — one feature or fix per PR. All Python code should pass existing tests (`pytest`) and follow the project's existing code style. For new features, include a test that covers the core behavior. For bug fixes, add a regression test if one does not already exist. By submitting a pull request you agree to license your contribution under the project's MIT License.
